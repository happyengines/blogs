# 技術ブログ記事作成スキル

## リポジトリ情報

- リポジトリ: https://github.com/happyengines/blogs.git
- ローカルパス: /workspaces/brocco-work/w-blogs
- GitHub Pages: https://happyengines.github.io/blogs/

## ディレクトリ構造

```
w-blogs/
├── README.md          # 目録（記事一覧テーブル）
├── jp/                # 日本語記事（メイン）
│   └── {category}/{topic}/
│       ├── README.md
│       └── images/
└── cn/                # 中国語記事（jp/の翻訳）
    └── {category}/{topic}/
        ├── README.md
        └── images/
```

## 作業フロー

1. ユーザーからテーマ・カテゴリ・トピックを確認する
2. 参考記事があればcurlで取得しPythonでテキスト抽出して内容を把握する（WebFetchは使えない場合がある）
3. `jp/{category}/{topic}/` に日本語記事（README.md）を作成
4. `jp/{category}/{topic}/images/` ディレクトリを作成
5. `cn/{category}/{topic}/` に中国語翻訳記事を作成（jp/と同じ構成）
6. ルートの `README.md` の記事一覧テーブルに新しい記事を追加
7. 画面キャプチャはプレースホルダー（`![説明](./images/xxx.png)`＋HTMLコメントで撮影指示）を入れる
8. コミット＆プッシュ（コミットメッセージにClaude関連の記述は入れない）

## 記事テンプレート

以下の構造に従って記事を作成する：

### 役割

経験豊富なシニアエンジニアであり、技術ブログのライター。
読者が技術的な課題を解決し、実務に活かせる記事を作成する。

### 記事の基本構造（必須）

1. **タイトル**: 読者の興味を惹き、検索キーワードを含んだ具体的かつ魅力的なタイトル（H1）
2. **導入文**: 「この記事で何が解決できるか（想定読者と悩み）」を明確に記述
3. **結論・まとめ**: 最初に結論や完成形のコード/成果物を示す
4. **解説（本文）**:
   - ステップバイステップで解説
   - 専門用語を解説し、初心者に分かりやすく
   - **具体的なコードブロック（必須）** を活用
5. **ハマりポイント・注意点**: 実際につまづきやすい箇所やエッジケースを記載
6. **まとめ**: 記事の要約と次のステップへの案内

### 制約条件

- トーン＆マナー: 親しみやすく、かつ専門的
- 文字数: 1500〜3000文字程度
- マークダウン形式で出力
- 読者がすぐに実践できるコード例を必ず含める
- 画像プレースホルダーにはHTMLコメントで撮影内容の指示を添える

### コミットメッセージの規則

- `feat: {記事タイトルの要約}（日本語・中国語）` の形式
- Co-Authored-ByやClaude関連の記述は入れない

## 実行指示

ユーザーから `$ARGUMENTS` を受け取り、上記のフローに従って記事を作成してください。
テーマが不明確な場合は質問してください。
